Microfinance Database System

ABC Microfinance Uganda Ltd — Centralised relational database, Bank of Uganda compliant
Stack

Overview

ABC Microfinance Uganda Ltd is a Tier 3 Microfinance Deposit-taking Institution licensed by the Bank of Uganda, serving over 12,000 clients across offices in Kampala, Jinja, and Mbarara. This project designed and implemented a centralised relational database system to replace their fragmented Excel-based records.

MySQL 8.0 PHP / Apache 3NF Normalization Role-Based Access Control CIA Triad Bank of Uganda Compliant

Problem

The organisation managed financial records for 12,000+ clients using Microsoft Excel spreadsheets and paper forms with no shared database or network connection between branches. Staff transferred data via USB drives and WhatsApp.

Data Issues
  • Duplicate client records across branches
  • Permanent data loss risk from USB failure
  • No real-time consolidated reporting
Operational Issues
  • Loan approvals delayed by days
  • Zero access controls on records
  • Non-compliant with Bank of Uganda audit requirements

Solution

A centralised MySQL database normalised to Third Normal Form (3NF), hosted at the Kampala head office and accessible to all 34 staff across three branches through a secure WAN connection. A PHP web application provides full CRUD operations without requiring SQL knowledge.

What I Did

Database Design
  • Requirements analysis with 6 stakeholder roles
  • Entity-Relationship diagram (6 entities)
  • Normalization from flat Excel to 3NF
  • Primary keys, foreign keys, CHECK constraints
Security
  • 4 role-based user accounts (Teller, Loan Officer, Branch Manager, IT Admin)
  • Segregation of duties — loan approval and disbursement by different roles
  • Soft deletion for 7-year audit compliance
SQL Development
  • Multi-table JOINs (4 tables, aliased)
  • Aggregates with CASE WHEN and GROUP BY
  • Subqueries (NOT IN, scalar)
  • 4 views for daily reporting
Application
  • Three-tier PHP/Apache web application
  • 6 CRUD screens for all staff roles
  • Dashboard with KPI cards and charts
  • Daily automated backup strategy

Key SQL — Active Loan Portfolio Query

-- Four-table JOIN: active loans with member, branch, approving officer and teller
SELECT m.FullName, b.BranchName,
       l.LoanAmount,
       l.LoanAmount - l.AmountRepaid AS Outstanding,
       s1.FullName AS ApprovedBy,
       s2.FullName AS DisbursedBy
FROM  Loan l
JOIN  Member m  ON l.MemberID   = m.MemberID
JOIN  Branch b  ON m.BranchID   = b.BranchID
JOIN  Staff  s1 ON l.ApprovedBy  = s1.StaffID
JOIN  Staff  s2 ON l.DisbursedBy = s2.StaffID
WHERE l.Status = 'Active';
